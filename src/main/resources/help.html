🎯 Überblick Kurzfassung

VEBO Lagersystem ist eine Desktop-Anwendung zur Verwaltung von Lagerbeständen, Artikeln, Bestellungen, Lieferanten und Abteilungen – entwickelt für VEBO Oensingen.

  • Effiziente Lagerverwaltung: Echtzeitübersicht über Bestände
  • Automatisierung: Periodische Lagerchecks & Warnungen
  • Benutzerfreundlichkeit: Moderne, konsistente UI mit Theme-/Font-Optionen
  • Nachvollziehbarkeit: Protokolle, Audit-Trails und Logs
  • Flexibilität: Anpassbare Einstellungen, QR-Workflow, Updates
Tipp: Für eine vollständige technische Dokumentation siehe die README im Repository.

✨ Hauptfunktionen Auszug

📦 Artikelverwaltung Core
  • Artikelstamm inkl. Artikelnummer, Details, Preise
  • Bestandsführung mit Min/Max, Kategorien, Lieferantenzuordnung
  • Bulk-Aktionen: Mehrfach löschen, Bestand anpassen, Export
  • QR-Codes: Generieren, Vorschau, PDF-Export, Daten abrufen
📋 Bestellungsverwaltung Workflow
  • Erstellen/Bearbeiten/Löschen, Status (Offen/Abgeschlossen)
  • Abschließen-Workflow, Filter & Suche
  • PDF-Export mit Layout + Protokoll inkl. Benutzer-Tracking
⚠️ Warnsystem & Scheduler Automatik
  • Automatische Bestandsprüfungen (Intervall konfigurierbar)
  • Warnungen bei Mindest-/kritischen Beständen
  • Ungelesene Warnungen werden hervorgehoben
📊 Logs, Import/Export & Backup Admin
  • Log-UI mit Kategorien, Datumsfilter, Export (PDF/CSV)
  • CSV Import/Export (Artikel, Lieferanten, Kunden, Bestellungen)
  • Startdaten-Import aus JSON-Ressourcen

📥 Installation Maven

Voraussetzungen: JDK 21+, Maven 3.6+, Git

git clone https://github.com/frame-dev/VeboLagerSystem.git
cd VeboLagerSystem
mvn clean install
mvn exec:java -Dexec.mainClass="ch.framedev.lagersystem.main.Main"

Alternative (JAR):

mvn clean package
                    java -jar target/VeboLagerSystem-0.3-TESTING.jar

🚀 Verwendung Basics

  • Erster Start: DB & Einstellungen werden initialisiert, optional Import von Startdaten
  • Artikel: hinzufügen/bearbeiten/löschen, Bulk-Aktionen, QR-Export
  • Bestellungen: anlegen, bearbeiten, abschließen, PDF-Export
  • Logs: filtern, suchen, exportieren

🏗️ Architektur Patterns

  • Singleton: Manager-Klassen
  • Observer: Theme-Updates für Fenster
  • Factory: UI-Komponenten
  • MVC: Trennung GUI / Business-Logic / Daten
Hinweis: Die vollständige Projektstruktur ist in der README dokumentiert.

⚙️ Konfiguration settings.properties

Die Einstellungen werden im App-Datenverzeichnis in settings.properties gespeichert.

# First run / Import
first-time=true
load-from-files=true

# Scheduler
stock_check_interval=30
enable_auto_stock_check=true

# QR-Code Import
enable_automatic_import_qrcode=true
qrcode_import_interval=10
server_url=https://framedev.ch/vebo/scans.json

# Theme / UI
dark_mode=false
table_font_size=16
font_style=Dialog

# Update-Manager
github-token=

💾 App-Datenverzeichnis Paths

  • Windows: %APPDATA%/VeboLagerSystem/
  • macOS: ~/Library/Application Support/VeboLagerSystem/
  • Linux: $XDG_DATA_HOME/VeboLagerSystem/ oder ~/.local/share/VeboLagerSystem/

📡 QR-Scan-Server (Optional) ScanServer

Optionaler kleiner HTTP-Server für QR-Scans (ScanServer).

mvn exec:java -Dexec.mainClass="ch.framedev.lagersystem.scan.ScanServer"
  • /scan?data=... (Formular + Speicherung)
  • /list (JSON aller Scans)
  • /latest (letzter Scan als JSON)
